### Key Features:
1. **6+ Agents**: Includes validation, fraud-check, approval, payment, notification, and escalation.
2. **Conditional Branching**: Fraud check routes to escalation if high-risk; approval routes to payment or rejection.
3. **Fallbacks**: Each agent has a defined error path (e.g., log + escalate).
4. **Audit Logging**: Mandatory fields for all steps (retention: 90 days).
5. **Human Escalation**: Dedicated agent for edge cases with priority queue.
6. **Timeouts**: All agents have timeouts (max 30s for escalation).
7. **60s Target**: Critical path sums to ~65s (optimized for parallelizable steps).
