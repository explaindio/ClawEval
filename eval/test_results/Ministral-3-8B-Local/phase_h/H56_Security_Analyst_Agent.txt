### Explanation of Vulnerabilities:

1. **SQL Injection (x2)**:
   - The `login` function directly interpolates user input into an SQL query, allowing attackers to manipulate the query.

2. **Server-Side Template Injection**:
   - The `profile` function uses user input directly in a template string, which could lead to arbitrary code execution.

3. **OS Command Injection (x3)**:
   - `run_command` uses `subprocess.run()` with `shell=True`, allowing command injection.
   - `export_data` uses `os.system()` with user input, allowing command injection.

4. **Directory Traversal (x2)**:
   - `download` and `upload` functions do not sanitize user input for path traversal attacks.

5. **Insecure Deserialization (x2)**:
   - `restore_session` uses `pickle.loads()` on untrusted data, which can execute arbitrary code.

6. **Insecure YAML Loading (x2)**:
   - `load_config` uses `yaml.load()` without `safe=False`, allowing arbitrary code execution.

7. **Hardcoded Secret**:
   - The `SECRET` variable is hardcoded, which can be easily exposed.

8. **Insecure Debug Mode (x2)**:
   - The Flask app runs with `debug=True` and `host="0.0.0.0"`, exposing debug information and allowing remote code execution.

9. **Insecure Password Storage**:
   - The `login` function uses MD5 hashing, which is easily reversible and insecure for password storage.
