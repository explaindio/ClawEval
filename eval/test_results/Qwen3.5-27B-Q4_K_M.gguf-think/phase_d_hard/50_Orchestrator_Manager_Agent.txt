# Orchestration Plan: Quarterly Business Review (QBR) Workflow

## 1. Execution DAG (Directed Acyclic Graph)

The workflow is designed to maximize parallelism while respecting data dependencies. Agent D is instantiated as three parallel sub-agents (`D_Fin`, `D_Mkt`, `D_Vis`) to handle specific sections concurrently.


**Dependencies:**

*   **Level 2:** B (Depends on A), E (Depends on A)

*   **Level 4:** F (Depends on All D outputs)

## 2. Critical Path Analysis

We calculate the minimum time required to complete the workflow under ideal conditions (no failures).

*   **Path 1 (Finance):** A (2m) → B (3m) → D_Fin (5m) → F (4m) = **14 mins**
*   **Path 2 (Visuals):** A (2m) → E (3m) → D_Vis (5m) → F (4m) = **14 mins**
*   **Path 3 (Market):** C (8m) → D_Mkt (5m) → F (4m) = **17 mins**

**Critical Path:** Path 3 (Market Chain).
**Minimum Completion Time:** **17 Minutes.**
**Budget Buffer:** 25m - 17m = **8 Minutes.**

This 8-minute buffer is crucial for handling retries without violating the 25-minute wall-clock constraint.

## 3. Failure Handling Strategy

Each agent has a specific failure protocol to ensure the QBR is delivered within the 25-minute budget.

| Agent | Failure Rate | Strategy | Max Retries | Fallback Action |
| :--- | :--- | :--- | :--- | :--- |
| **A (Data)** | 0% | N/A | 0 | N/A |
| **B (Finance)** | 5% | **Immediate Retry** | 1 | If fail twice: Skip Financial Section. D writes "Data Unavailable". |
| **C (Market)** | 20% | **Immediate Retry** | 1 | If fail twice: Trigger **Plan B** (See Section 4). |
| **D (Writer)** | Low | **Immediate Retry** | 1 | If fail twice: Skip specific section. |
| **E (Designer)** | Low | **Immediate Retry** | 1 | If fail twice: Skip Visuals. D writes "Charts Pending". |
| **F (Reviewer)** | Low | **Immediate Retry** | 1 | If fail twice: Auto-approve (Emergency Bypass). |

**Orchestration Logic:**
1.  **Timeouts:** All agents have a hard timeout equal to their standard duration + 10% (e.g., Agent C has a 9-minute timeout).
2.  **State Management:** The orchestrator tracks the state of every task. If a task fails, it logs the error and triggers the retry logic immediately.
3.  **Synchronization:** Agent F will not start until all D tasks are either **Completed** or **Marked as Skipped**.

## 4. Plan B: Agent C Double Failure Protocol

If Agent C fails twice (Probability: $0.2 \times 0.2 = 4\%$), the Market Data is unavailable. The QBR must still be produced.

**Trigger Condition:** Agent C fails at T=8m AND fails again at T=16m.

**Execution Steps:**
1.  **Abort C:** Stop further retries on Agent C to save time.
2.  **Notify D:** Send a signal to `D_Mkt` to cancel the research task.
3.  **Fallback Content:** `D_Mkt` generates a static placeholder section:
    > *"Note: Market trend analysis could not be completed due to data source unavailability. Please refer to the previous quarter's report for historical context."*
4.  **Update Dependencies:** Agent F is notified that `D_Mkt` is "Complete (Fallback)".
5.  **Proceed:** Agent F proceeds with the review using available sections (Finance + Visuals + Fallback).

**Time Impact:**
*   Without Plan B, we would wait until T=16m to realize C failed.
*   With Plan B, we proceed immediately.
*   Since `D_Fin` and `D_Vis` finish at T=10m, Agent F can start at **T=10m** (instead of waiting for C).
*   **Total Time:** 14 Minutes. (Well within budget).
