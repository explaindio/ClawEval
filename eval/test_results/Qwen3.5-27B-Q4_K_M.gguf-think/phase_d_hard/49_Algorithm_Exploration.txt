## PROMPT:
Design an algorithm for this problem. There's an O(n) solution but most people give O(n log n) or O(n²). Find the optimal one.

Problem: Given an array of n integers, find the maximum difference arr[j] - arr[i] such that j > i (the larger element must come AFTER the smaller element). If no positive difference exists, return 0.

Example: [2, 3, 10, 6, 4, 8, 1] → Answer: 8 (10-2)
Example: [7, 9, 5, 6, 3, 2] → Answer: 2 (9-7, not 5-2 because 5 comes after 9)
Example: [10, 8, 6, 4, 2] → Answer: 0 (strictly decreasing)

Requirements:
1. Give the naive O(n²) solution and explain why it's slow
2. Give an O(n log n) solution using a common technique
3. Give the OPTIMAL O(n) solution — track minimum so far
4. Prove the O(n) solution is correct with a formal argument
5. Handle edge cases: empty array, single element, all same values, all negative
6. Provide Python implementation of the O(n) solution with tests

### 1. The Naive $O(n^2)$ Solution

**Logic:**
The most straightforward approach is to check every possible pair $(i, j)$ where $i < j$. We calculate the difference `arr[j] - arr[i]` and keep track of the maximum value found.

**Algorithm:**
```python
def max_diff_naive(arr):
    n = len(arr)
    if n < 2:
        return 0
    
    max_diff = 0
    for i in range(n):
        for j in range(i + 1, n):
            diff = arr[j] - arr[i]
            if diff > max_diff:
                max_diff = diff
    return max_diff
```

**Why it's slow:**
This solution uses two nested loops. The outer loop runs $n$ times, and the inner loop runs roughly $n/2$ times on average.
*   **Time Complexity:** $O(n^2)$.
*   **Performance:** For an array of size $10^5$, this performs approximately $5 \times 10^9$ operations. Most online judges allow $\approx 10^8$ operations per second. This will result in a **Time Limit Exceeded (TLE)** error for large inputs.

---

### 2. The $O(n \log n)$ Solution (Divide and Conquer)

**Logic:**
A common mistake is to sort the array ($O(n \log n)$) and take `max - min`. However, this violates the constraint $j > i$ (the larger element must appear *after* the smaller one).

To achieve a valid $O(n \log n)$ solution, we use **Divide and Conquer** (similar to Merge Sort). We split the array into two halves. The maximum difference is either:
1.  Entirely in the left half.
2.  Entirely in the right half.
3.  Crossing the midpoint (minimum in the left half, maximum in the right half).

**Algorithm:**
```python
def max_diff_divide_conquer(arr):
    def solve(left, right):
        if left >= right:
            return 0
        
        mid = (left + right) // 2
        
        # 1. Max diff in left half
        max_left = solve(left, mid)
        # 2. Max diff in right half
        max_right = solve(mid + 1, right)
        
        # 3. Max diff crossing the midpoint
        # Find min in left half and max in right half
        min_left = min(arr[left : mid + 1])
        max_right_part = max(arr[mid + 1 : right + 1])
        max_cross = max_right_part - min_left
        
        return max(max_left, max_right, max_cross)

    if len(arr) < 2:
        return 0
    return solve(0, len(arr) - 1)
```

**Complexity:**
*   **Time:** $T(n) = 2T(n/2) + O(n)$. By the Master Theorem, this resolves to **$O(n \log n)$**.
*   **Space:** $O(\log n)$ for the recursion stack.

### 3. The OPTIMAL $O(n)$ Solution (Greedy / One-Pass)

**Logic:**
We do not need to check every pair. For any element `arr[j]`, to maximize `arr[j] - arr[i]`, we simply need the **smallest `arr[i]` that appeared before `j`**.

We can maintain a variable `min_so_far` as we iterate through the array. At each step, we calculate the potential profit using the current element and the minimum seen so far. We update the global maximum if this potential profit is higher.

**Algorithm:**
1.  Initialize `min_so_far` to the first element.
2.  Initialize `max_diff` to 0.
3.  Iterate through the array starting from the second element.
4.  Update `max_diff` = `max(max_diff, current_element - min_so_far)`.
5.  Update `min_so_far` = `min(min_so_far, current_element)`.
6.  Return `max_diff`.

**Complexity:**
*   **Time:** **$O(n)$** (Single pass).
*   **Space:** **$O(1)$** (Two variables).

---

### 4. Proof of Correctness (Formal Argument)

We prove the correctness of the $O(n)$ solution using **Mathematical Induction** on the loop index $k$.

**Invariant:**
At the end of iteration $k$ (where we have processed `arr[0]` through `arr[k]`):
1.  `min_so_far` holds $\min(arr[0], arr[1], \dots, arr[k])$.
2.  `max_diff` holds $\max_{0 \le i < j \le k} (arr[j] - arr[i])$.

**Base Case ($k=1$):**
*   We initialize `min_so_far = arr[0]`.
*   We process `arr[1]`.
*   `max_diff` becomes $\max(0, arr[1] - arr[0])$.
*   `min_so_far` becomes $\min(arr[0], arr[1])$.
*   The invariant holds for the range $[0, 1]$.

**Inductive Step:**
Assume the invariant holds for iteration $k-1$. We now process element $arr[k]$.
*   **Update `min_so_far`:** The new minimum in range $[0, k]$ is either the old minimum (from $[0, k-1]$) or the new element $arr[k]$. The code `min_so_far = min(min_so_far, arr[k])` correctly maintains this.
*   **Update `max_diff`:** The maximum difference in range $[0, k]$ is either:
    1.  A pair entirely within $[0, k-1]$ (covered by the previous `max_diff`).
    2.  A pair ending at $k$, i.e., $arr[k] - arr[i]$ where $0 \le i < k$.
    To maximize case 2, we need the smallest $arr[i]$ in $[0, k-1]$, which is exactly `min_so_far` (from the inductive hypothesis).
    The code `max_diff = max(max_diff, arr[k] - min_so_far)` correctly captures the maximum of case 1 and case 2.

**Conclusion:**
Since the invariant holds after the final iteration ($k = n-1$), `max_diff` contains the maximum difference for the entire array.

---

### 5. Edge Case Handling

The $O(n)$ solution naturally handles these cases:

1.  **Empty Array:** Check `len(arr) < 2` at the start. Return 0.
2.  **Single Element:** Loop range `range(1, 1)` is empty. Returns initial `max_diff` (0).
3.  **All Same Values:** `arr[j] - arr[i]` will always be 0. `max_diff` remains 0.
4.  **All Negative:** `arr[j] - arr[i]` might be negative. Since we initialize `max_diff = 0` and take `max`, we return 0 (as per requirement "If no positive difference exists, return 0").
5.  **Strictly Decreasing:** `min_so_far` updates constantly. `arr[j] - min_so_far` will always be $\le 0$. `max_diff` remains 0.

### 6. Python Implementation & Tests

```python
import unittest

def max_profit(arr):
    """
    Finds the maximum difference arr[j] - arr[i] such that j > i.
    Returns 0 if no positive difference exists.
    Time Complexity: O(n)
    Space Complexity: O(1)
    """
    if len(arr) < 2:
        return 0
    
    min_so_far = arr[0]
    max_diff = 0
    
    for i in range(1, len(arr)):
        # Calculate potential profit with current element
        current_diff = arr[i] - min_so_far
        
        # Update global max if current is better
        if current_diff > max_diff:
            max_diff = current_diff
            
        # Update minimum seen so far
        if arr[i] < min_so_far:
            min_so_far = arr[i]
            
    return max_diff

class TestMaxProfit(unittest.TestCase):
    
    def test_example_1(self):
        # [2, 3, 10, 6, 4, 8, 1] -> 10 - 2 = 8
        self.assertEqual(max_profit([2, 3, 10, 6, 4, 8, 1]), 8)
        
    def test_example_2(self):
        # [7, 9, 5, 6, 3, 2] -> 9 - 7 = 2
        self.assertEqual(max_profit([7, 9, 5, 6, 3, 2]), 2)
        
    def test_example_3(self):
        # [10, 8, 6, 4, 2] -> 0 (Strictly decreasing)
        self.assertEqual(max_profit([10, 8, 6, 4, 2]), 0)
        
    def test_empty_array(self):
        self.assertEqual(max_profit([]), 0)
        
    def test_single_element(self):
        self.assertEqual(max_profit([5]), 0)
        
    def test_all_same(self):
        self.assertEqual(max_profit([5, 5, 5, 5]), 0)
        
    def test_all_negative(self):
        # [-5, -2, -8, -1] -> -1 - (-5) = 4
        self.assertEqual(max_profit([-5, -2, -8, -1]), 4)
        
    def test_negative_decreasing(self):
        # [-1, -2, -3] -> 0
        self.assertEqual(max_profit([-1, -2, -3]), 0)
        
    def test_large_increase(self):
        # [1, 100] -> 99
        self.assertEqual(max_profit([1, 100]), 99)

if __name__ == '__main__':
    unittest.main()
```
