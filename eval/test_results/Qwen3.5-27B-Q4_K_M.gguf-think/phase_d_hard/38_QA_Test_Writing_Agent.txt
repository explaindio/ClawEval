## 🐛 BUG EXPLANATION

**The Bug:** The function **does not validate page numbers**, allowing `page=0` and negative page values.

### What Happens:

1. **Page 0**: 
   - `start = (0-1) * 10 = -10`
   - `end = -10 + 10 = 0`
   - `items[-10:0]` returns empty list (Python slice behavior)
   - `has_next = 0 < total_pages = True` ❌ **WRONG** - page 0 shouldn't exist
   - `has_prev = 0 > 1 = False`

2. **Negative Page (-1)**:
   - `start = (-1-1) * 10 = -20`
   - `end = -20 + 10 = -10`
   - `items[-20:-10]` returns actual items from end of list ❌ **WRONG**

### Why It's Subtle:

- The function doesn't crash - it silently accepts invalid input
- Python's slice behavior masks the issue (negative indices work)
- Metadata becomes inconsistent with actual data returned
- Only caught when testing edge cases like page=0

### Fix:

```python
def paginate(items, page, per_page=10):
    """Return a page of items with metadata."""
    if page < 1:
        raise ValueError(f"Page must be >= 1, got {page}")
    
    total = len(items)
    total_pages = total // per_page + (1 if total % per_page else 0)
    
    start = (page - 1) * per_page
    end = start + per_page
    
    return {
        "items": items[start:end],
        "page": page,
        "per_page": per_page,
        "total": total,
        "total_pages": total_pages,
        "has_next": page < total_pages,
        "has_prev": page > 1
    }
```

Run with: `pytest -v` - tests 7 and 8 will fail, proving the bug exists.
