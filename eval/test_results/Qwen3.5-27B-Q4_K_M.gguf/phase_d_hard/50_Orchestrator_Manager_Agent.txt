## 4. Plan B: Agent C Double Failure Protocol

If Agent C fails the initial run **and** the retry run, the following protocol activates to ensure the QBR is delivered within the 25-minute limit.

1.  **Trigger:** At T=16 (after 2nd failure), the Orchestrator marks `C_Status` as `CRITICAL_FAIL`.
2.  **Input Generation:** Instead of waiting for C, the Orchestrator generates a static JSON payload:
    ```json
    {
      "status": "MISSING",
      "message": "Market trend data unavailable due to system error.",
      "recommendation": "Refer to previous quarter's market analysis."
    }
    ```
3.  **Agent D Instruction:** `D_Mkt` is instructed to write a "Market Overview" section based on the static payload. It must explicitly state that live data is missing.
4.  **Agent F Instruction:** `F` is instructed to add a **Red Banner** to the final document header: *"Warning: Market Data Section is based on fallback logic."*
5.  **Time Budget:** This path consumes exactly 25 minutes (8m + 8m + 5m + 4m). No further retries are permitted.
