## 🐛 BUG EXPLANATION

### The Bug: **No Input Validation for Page Parameter**

The function accepts **page=0** and **negative page numbers** without validation, leading to incorrect behavior:

#### Case 1: `page=0`
```python
start = (0 - 1) * 10 = -10
end = -10 + 10 = 0
items[-10:0]  # Returns empty list (unexpected)
has_next = 0 < total_pages  # Returns True (WRONG!)
has_prev = 0 > 1  # Returns False (correct)
```

**Problem:** `has_next` is `True` even though page 0 is invalid.

#### Case 2: `page=-1`
```python
start = (-1 - 1) * 10 = -20
end = -20 + 10 = -10
items[-20:-10]  # Returns LAST 10 items (WRONG!)
```

**Problem:** Negative pages return the wrong items (from the end of the list).

### Why Tests Fail

The tests `test_bug_page_zero` and `test_bug_negative_page` will **FAIL** because:
1. `page=0` returns empty items but `has_next=True` (inconsistent)
2. `page=-1` returns actual items (the last 10), not empty

### Fix Required

Add input validation at the start of the function:

```python
def paginate(items, page, per_page=10):
    if page < 1:
        raise ValueError("page must be >= 1")
    if per_page < 1:
        raise ValueError("per_page must be >= 1")
    # ... rest of function
```

This ensures only valid page numbers are accepted, preventing the subtle bug from causing incorrect pagination behavior.
