## PROMPT:
Write tests for this pagination function. It has a SUBTLE BUG — your tests must catch it. 

```python
def paginate(items, page, per_page=10):
    """Return a page of items with metadata."""
    total = len(items)
    total_pages = total // per_page + (1 if total % per_page else 0)
    
    start = (page - 1) * per_page
    end = start + per_page
    
    return {
        "items": items[start:end],
        "page": page,
        "per_page": per_page,
        "total": total,
        "total_pages": total_pages,
        "has_next": page < total_pages,
        "has_prev": page > 1
    }
```

Write pytest tests that:
1. Test normal pagination (100 items, page 1, per_page 10)
2. Test last page with partial results
3. Test empty list
4. Test page beyond range
5. Test per_page larger than total items
6. Test exactly divisible (e.g., 20 items, per_page 10)
7. FIND THE BUG (hint: what happens with page=0? or negative page?)

One of your tests MUST fail, proving the bug exists. Explain what the bug is.

## The Bug

**The pagination function accepts invalid page numbers (0 or negative) without validation.**

### What happens:
- When `page=0`: `start = (0 - 1) * per_page = -per_page`
- Python slice `items[-per_page:0]` uses negative indexing
- This returns empty or unexpected results
- But the metadata still reports `page=0`, `has_next=True`, etc.

### Why it's dangerous:
1. **Silent failure**: Returns empty results instead of raising an error
2. **Misleading metadata**: Claims page 0 exists when it shouldn't
3. **Inconsistent behavior**: `has_next=True` but no items returned
4. **Security risk**: Could be exploited to bypass pagination logic

### Fix:
Add validation at the start of the function:
```python
if page < 1:
    raise ValueError("Page must be at least 1")
```

The test `test_invalid_page_zero` will fail because it expects the bug to exist (asserts page=0 is accepted). In a production fix, this test should expect a `ValueError` instead.
