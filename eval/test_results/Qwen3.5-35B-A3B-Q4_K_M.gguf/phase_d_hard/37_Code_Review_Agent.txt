### 🔴 Critical Bug (Hard to Spot): Race Condition Causing Money Loss

**Problem:** The `transfer()` method is completely unprotected by locks. The balance check and update are not atomic, allowing multiple threads to pass the balance check and both deduct funds. This can result in overdrafts and money creation.

**Example Scenario:**
```
Account1: $100, Account2: $100
Thread A: transfer(acc1, acc2, $80)
Thread B: transfer(acc2, acc1, $80)

1. Both threads check balance: $100 >= $80 → both pass
2. Thread A deducts $80 from acc1 → acc1 = $20
3. Thread B deducts $80 from acc2 → acc2 = $20
4. Thread A adds $80 to acc2 → acc2 = $100
5. Thread B adds $80 to acc1 → acc1 = $100

Result: $160 transferred, but only $200 existed. $20 was created out of thin air.
```

**Fix:** Use proper locking on both accounts, and ensure the balance check and update are atomic.

---

### 🔵 5 Obvious Issues

1. **Using `float` for Money**
   - `float` introduces precision errors in financial calculations.
   - **Fix:** Use `Decimal` for all monetary values.

   ```python
   from decimal import Decimal
   self.balance = Decimal(str(balance))
   ```

2. **`transfer()` Method Doesn't Use Locks**
   - The `self.lock` is created but never used.
   - **Fix:** Acquire locks before modifying balances.

   ```python
   with self.lock:
       with target_account.lock:
           # perform transfer
   ```

3. **No Lock on Target Account**
   - Transferring between accounts without locking both can lead to race conditions or deadlocks.
   - **Fix:** Lock both accounts in a consistent order (e.g., by account ID) to avoid deadlocks.

4. **Inconsistent Balance Reporting**
   - `get_balance()` returns a rounded value, but internal balance is unrounded.
   - **Fix:** Store and return balances consistently using `Decimal`.

   ```python
   def get_balance(self):
       return self.balance.quantize(Decimal('0.01'))
   ```

5. **Improper Amount Handling**
   - `amount` is converted to `float`, losing precision.
   - **Fix:** Use `Decimal` for all amounts.

   ```python
   amount = Decimal(str(amount))
   ```

import threading
from decimal import Decimal, ROUND_HALF_UP

class BankAccount:
    def __init__(self, account_id, balance):
        self.account_id = account_id
        self.balance = Decimal(str(balance))
        self.lock = threading.Lock()
        self.transactions = []

    def transfer(self, target_account, amount):
        amount = Decimal(str(amount))
        if amount <= 0:
            raise ValueError("Amount must be positive")

        # Lock both accounts in consistent order to prevent deadlock
        first, second = (self, target_account) if self.account_id < target_account.account_id \
                        else (target_account, self)

        with first.lock:
            with second.lock:
                if self.balance >= amount:
                    self.balance -= amount
                    target_account.balance += amount

                    self.transactions.append({
                        'type': 'transfer',
                        'amount': amount,
                        'to': target_account.account_id,
                        'balance_after': self.balance
                    })
                    return True
        return False

    def get_balance(self):
        return self.balance.quantize(Decimal('0.01'))
