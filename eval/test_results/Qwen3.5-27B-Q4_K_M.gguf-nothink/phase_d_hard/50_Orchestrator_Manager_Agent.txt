## 2. Critical Path Analysis

We calculate the minimum time required in the "Happy Path" (no failures) and the "Failure Path" (Agent C fails twice).

### Scenario A: Happy Path (No Failures)
1.  **Start**: 0m
2.  **Parallel Start**:
    *   Agent A: 0m -> 2m
    *   Agent C: 0m -> 8m
3.  **Secondary Parallel** (Starts at 2m):
    *   Agent B: 2m -> 5m
    *   Agent E: 2m -> 5m
4.  **Writing Phase** (Starts when dependencies are met):
    *   D1 (Finance): Starts at 5m (B done) -> Ends 10m.
    *   D3 (Data/Vis): Starts at 5m (E done) -> Ends 10m.
    *   D2 (Market): Starts at 8m (C done) -> Ends 13m.
    *   *Constraint*: All D agents must finish before F starts. Max time = 13m.
5.  **Review**:
    *   Agent F: Starts at 13m -> Ends 17m.

**Minimum Time (Happy Path): 17 Minutes.**
*Buffer available: 8 minutes.*

### Scenario B: Agent C Fails Twice (Worst Case)
*Constraint: Agent C takes 8m. If it fails, we retry. We allow 2 retries max.*
1.  **Start**: 0m
2.  **Agent C Attempt 1**: 0m -> 8m (FAIL).
3.  **Agent C Attempt 2**: 8m -> 16m (FAIL).
4.  **Fallback Trigger**: At 16m, we skip D2 and generate a "Missing Data" placeholder.
5.  **Other Agents**:
    *   A, B, E, D1, D3 finish by 10m (as calculated above).
    *   They wait for the C branch to resolve (either success or fallback).
6.  **Writing Phase Resolution**:
    *   D1, D3 finished at 10m.
    *   D2 (Fallback): Instant generation of "Market Data Unavailable" note (0m).
    *   *Constraint*: F starts at max(10m, 16m) = 16m.
7.  **Review**:
    *   Agent F: Starts at 16m -> Ends 20m.

**Maximum Time (C Fails Twice): 20 Minutes.**
*Buffer available: 5 minutes.*
*Result: Within the 25-minute budget.*

## 3. Failure Handling Strategy

| Agent | Failure Rate | Strategy | Action on Failure |
| :--- | :--- | :--- | : |
| **Agent A** | 0% | None | N/A |
| **Agent B** | 5% | **Retry (1x)** | If B fails, retry immediately. If 2nd fail, use "Financial Data Pending" placeholder. |
| **Agent C** | 20% | **Retry (2x) + Fallback** | Retry up to 2 times. If 3rd attempt fails (or 2nd if configured as max 2 attempts), trigger **Plan B** (Skip D2, insert disclaimer). |
| **Agent D** | Low | **Retry (1x)** | If a specific section fails, retry once. If fail again, insert "Section Generation Error" placeholder. |
| **Agent E** | Low | **Retry (1x)** | If visuals fail, insert "Chart Generation Error" placeholder. |
| **Agent F** | Low | **Retry (1x)** | If review fails, retry once. If fail, auto-approve with "Review Pending" flag. |

**Specific Logic for Agent C (The Critical Risk):**
*   **Timeout**: Set strict timeout of 8 minutes per attempt.
*   **Retry Logic**:
    *   Attempt 1: Run 0m-8m.
    *   Attempt 2: Run 8m-16m.
    *   Attempt 3: **ABORT**. Do not run a 3rd attempt (would push timeline to 24m+ leaving no buffer for F).
*   **Fallback**: If 2 attempts fail, the orchestrator injects a pre-written JSON object: `{ "status": "missing", "note": "Market data unavailable due to service outage. Analysis pending." }` into the input for Agent D2.

## 4. Plan B: Agent C Fails Twice

If Agent C fails twice (consuming 16 minutes), the workflow enters **Plan B** mode:

1.  **Trigger**: At T=16m, the orchestrator detects the second failure.
2.  **Action**:
    *   Cancel any pending 3rd attempt for Agent C.
    *   Signal Agent D2 to execute a "Fallback Mode".
    *   **D2 Fallback Logic**: Instead of writing a full market analysis, D2 generates a standardized "Data Unavailable" section (takes <1 min).
3.  **Impact on Timeline**:
    *   Agents D1 and D3 have been idle since T=10m.
    *   Agent F waits until T=16m (when D2 finishes its fallback).
    *   Agent F runs T=16m to T=20m.
4.  **Output**: The QBR is generated with a prominent disclaimer in the Market section.
5.  **Post-Processing**: A separate async task is triggered to retry Agent C later (outside the 25m budget) to update the document if possible, but the immediate deliverable is met.
