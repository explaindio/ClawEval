## PROMPT:
Design an algorithm for this problem. There's an O(n) solution but most people give O(n log n) or O(n²). Find the optimal one.

Problem: Given an array of n integers, find the maximum difference arr[j] - arr[i] such that j > i (the larger element must come AFTER the smaller element). If no positive difference exists, return 0.

Example: [2, 3, 10, 6, 4, 8, 1] → Answer: 8 (10-2)
Example: [7, 9, 5, 6, 3, 2] → Answer: 2 (9-7, not 5-2 because 5 comes after 9)
Example: [10, 8, 6, 4, 2] → Answer: 0 (strictly decreasing)

Requirements:
1. Give the naive O(n²) solution and explain why it's slow
2. Give an O(n log n) solution using a common technique
3. Give the OPTIMAL O(n) solution — track minimum so far
4. Prove the O(n) solution is correct with a formal argument
5. Handle edge cases: empty array, single element, all same values, all negative
6. Provide Python implementation of the O(n) solution with tests

## RESPONSE:
## 1.  Naïve \(O(n^2)\) solution  

```text
maxDiff = 0
for i = 0 … n‑2:
    for j = i+1 … n‑1:
        diff = arr[j] – arr[i]
        if diff > maxDiff:
            maxDiff = diff
return maxDiff
```

*Why it is slow*  
The inner loop runs for every pair \((i,j)\) with \(j>i\).  
There are \(\frac{n(n-1)}{2}\) such pairs, so the algorithm performs
\(Θ(n^2)\) subtractions and comparisons.  
For \(n = 10^6\) this is about \(5·10^{11}\) operations – far too many for
any realistic run‑time.

--------------------------------------------------------------------

## 2.  \(O(n \log n)\) solution – “divide‑and‑conquer” / segment‑tree

The idea is to ask: *for a fixed left index \(i\), what is the maximum
value in the suffix \(i+1 … n-1\)?*  
If we can answer that query in \(O(\log n)\) time, we can scan all
\(i\) in linear time and still keep the total complexity at
\(O(n \log n)\).

### 2.1  Build a segment tree for range‑maximum

* Build: \(O(n)\)  
* Query maximum in a range \([l,r]\): \(O(\log n)\)

### 2.2  Algorithm

```text
build segment tree T over arr
maxDiff = 0
for i = 0 … n-2:
    maxAfter = T.query(i+1, n-1)      // maximum in suffix
    diff = maxAfter – arr[i]
    if diff > maxDiff:
        maxDiff = diff
return maxDiff
```

*Complexity*  
Building the tree: \(O(n)\)  
\(n-1\) queries, each \(O(\log n)\) → \(O(n \log n)\) total.  
Space: \(O(n)\) for the tree.

--------------------------------------------------------------------

## 3.  Optimal \(O(n)\) solution – “track the minimum so far”

The key observation:

> For a fixed right index \(j\), the best left index \(i<j\) that maximises
> \(arr[j]-arr[i]\) is the **minimum element among all indices < j**.

Therefore we can scan the array once, keep the smallest value seen so
far, and compute the difference with the current element.

```text
maxDiff = 0
minSoFar = +∞
for x in arr:
    if x - minSoFar > maxDiff:
        maxDiff = x - minSoFar
    if x < minSoFar:
        minSoFar = x
return maxDiff
```

*Why it works*  
At step \(j\) we have already seen all elements \(arr[0…j-1]\).  
`minSoFar` is the smallest of them, so `x - minSoFar` is the largest
possible difference ending at index \(j\).  
We keep the best of all such differences.

*Complexity*  
One pass, constant work per element → \(O(n)\) time, \(O(1)\) extra
space.

--------------------------------------------------------------------

## 4.  Proof of correctness (formal)

We prove that the algorithm returns the maximum possible
\(arr[j]-arr[i]\) with \(j>i\).

### Lemma 1  
Let \(j\) be any index. Let \(m_j = \min\{arr[i] \mid 0 \le i < j\}\).
Then for every \(i<j\),

\[
arr[j] - arr[i] \le arr[j] - m_j .
\]

*Proof.*  
By definition of \(m_j\), \(m_j \le arr[i]\). Subtracting \(arr[i]\) from
\(arr[j]\) gives a value no larger than subtracting the smaller \(m_j\).
∎

### Lemma 2  
During the scan, after processing element \(arr[j]\), the variable
`maxDiff` equals

\[
\max_{0 \le k \le j} \bigl( arr[k] - \min\{arr[i] \mid 0 \le i < k\} \bigr).
\]

*Proof by induction on \(j\).*  

*Base \(j=0\).*  
No pair exists yet, `maxDiff` is initialised to 0, which equals the
empty maximum.

*Induction step.*  
Assume the lemma holds after processing \(j-1\).  
When processing \(arr[j]\):

1. `minSoFar` equals \(\min\{arr[i] \mid 0 \le i < j\}\) by induction
   (it was updated with every element seen).
2. We compute `diff = arr[j] - minSoFar`, which by Lemma 1 is the
   largest difference ending at \(j\).
3. `maxDiff` is updated with `max(maxDiff, diff)`, so after the update
   it equals the maximum of all differences ending at indices
   \(\le j\).

Thus the lemma holds for \(j\). ∎

### Theorem  
After the loop terminates, `maxDiff` equals

\[
\max_{0 \le i < j < n} (arr[j] - arr[i]).
\]

*Proof.*  
By Lemma 2 with \(j = n-1\), `maxDiff` is the maximum over all
indices \(k\) of the largest difference ending at \(k\).  
Every valid pair \((i,j)\) is considered when \(k=j\); hence the
maximum over all pairs is exactly the value stored in `maxDiff`. ∎

--------------------------------------------------------------------

## 5.  Edge‑case handling

| Case | Expected result | Reason |
|------|-----------------|--------|
| Empty array (`[]`) | `0` | No pair exists. |
| One element (`[x]`) | `0` | No pair exists. |
| All equal (`[5,5,5]`) | `0` | All differences are zero. |
| All negative (`[-5,-3,-1]`) | `0` | Largest difference is negative; we return `0`. |
| Mixed signs (`[-2, 0, 3]`) | `5` | \(3 - (-2) = 5\). |
| Strictly decreasing (`[10,8,6]`) | `0` | No positive difference. |

The algorithm naturally returns `0` in all these situations because
`maxDiff` starts at `0` and only increases when a positive difference
is found.

--------------------------------------------------------------------

## 6.  Python implementation with unit tests

```python
from typing import List
import unittest
import random

def max_difference(arr: List[int]) -> int:
    """
    Return the maximum arr[j] - arr[i] with j > i.
    If no positive difference exists, return 0.
    """
    max_diff = 0
    min_so_far = float('inf')
    for x in arr:
        # potential difference ending at current element
        diff = x - min_so_far
        if diff > max_diff:
            max_diff = diff
        # update the running minimum
        if x < min_so_far:
            min_so_far = x
    return max_diff


class TestMaxDifference(unittest.TestCase):
    def test_examples(self):
        self.assertEqual(max_difference([2, 3, 10, 6, 4, 8, 1]), 8)
        self.assertEqual(max_difference([7, 9, 5, 6, 3, 2]), 2)
        self.assertEqual(max_difference([10, 8, 6, 4, 2]), 0)

    def test_edge_cases(self):
        self.assertEqual(max_difference([]), 0)
        self.assertEqual(max_difference([5]), 0)
        self.assertEqual(max_difference([5, 5, 5]), 0)
        self.assertEqual(max_difference([-5, -3, -1]), 0)
        self.assertEqual(max_difference([-2, 0, 3]), 5)
        self.assertEqual(max_difference([3, 2, 1, 0]), 0)

    def test_random(self):
        for _ in range(1000):
            n = random.randint(0, 20)
            arr = [random.randint(-10, 10) for _ in range(n)]
            # brute force
            best = 0
            for i in range(n):
                for j in range(i+1, n):
                    best = max(best, arr[j] - arr[i])
            self.assertEqual(max_difference(arr), best)

if __name__ == "__main__":
    unittest.main()
```

The tests cover the provided examples, all edge cases, and a large
number of random arrays to confirm that the implementation matches the
brute‑force result. The algorithm runs in linear time and constant
extra space, satisfying the optimal requirement.
