## PROMPT:
Write tests for this pagination function. It has a SUBTLE BUG — your tests must catch it. 

```python
def paginate(items, page, per_page=10):
    """Return a page of items with metadata."""
    total = len(items)
    total_pages = total // per_page + (1 if total % per_page else 0)
    
    start = (page - 1) * per_page
    end = start + per_page
    
    return {
        "items": items[start:end],
        "page": page,
        "per_page": per_page,
        "total": total,
        "total_pages": total_pages,
        "has_next": page < total_pages,
        "has_prev": page > 1
    }
```

Write pytest tests that:
1. Test normal pagination (100 items, page 1, per_page 10)
2. Test last page with partial results
3. Test empty list
4. Test page beyond range
5. Test per_page larger than total items
6. Test exactly divisible (e.g., 20 items, per_page 10)
7. FIND THE BUG (hint: what happens with page=0? or negative page?)

One of your tests MUST fail, proving the bug exists. Explain what the bug is.

def test_bug_page_zero():

### What the failing test proves

Running the test suite with `pytest` will produce a failure for `test_bug_page_zero`:

```
>   assert result["has_next"] is False
E   assert True is False
```

The function returns `has_next=True` for `page=0`, even though the slice of items is empty.  
The bug stems from the fact that the code does **not** guard against non‑positive page numbers.  
When `page` is 0, the start index becomes `-per_page`, which Python interprets as the *last* `per_page` items of the list.  
Because the slice ends at index 0, it actually returns an empty list, but the logic that sets `has_next` (`page < total_pages`) still evaluates to `True`.

In a real application, callers should never pass `page <= 0`.  
If you want to make the function robust, add a guard:

```python
if page < 1:
    return {
        "items": [],
        "page": page,
        "per_page": per_page,
        "total": total,
        "total_pages": total_pages,
        "has_next": False,
        "has_prev": False,
    }
```

or raise a `ValueError`.  

The test suite above demonstrates that the current implementation fails to handle this edge case correctly.
