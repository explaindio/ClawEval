## Key Semantic Choices Explained

| Element | Rationale |
|---------|-----------|
| `<header>` + `<nav>` | Primary navigation landmark with proper labeling |
| `<main>` | Contains all primary content, one per page |
| `<section>` with `aria-labelledby` | Each section has a heading ID reference for screen readers |
| `<article>` | Used for self-contained feature cards and pricing tiers |
| `<blockquote>` + `<cite>` | Proper semantic markup for testimonials |
| `<details>` + `<summary>` | Native HTML accordion without needing JavaScript |
| `<aside>` | Trust badges are tangentially related to main hero content |
| `<footer>` within `<blockquote>` | Correct citation structure for testimonials |
| `role="button"` | Links styled as buttons are semantically announced correctly |
| `aria-label` | Provides context for icon-only links and navigation regions |

This structure is fully accessible, SEO-friendly, and ready for CSS styling and JavaScript enhancement.
