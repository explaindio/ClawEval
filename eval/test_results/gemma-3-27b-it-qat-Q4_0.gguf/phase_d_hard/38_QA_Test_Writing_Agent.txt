**Explanation of the Bug:**

The bug lies in how the `start` index is calculated: `start = (page - 1) * per_page`.  When `page` is 0, `start` becomes `-per_page`.  Python list slicing with a negative start index counts from the end of the list.  This results in the function returning the last `per_page` elements of the list, which is not the intended behavior for page 0.  Pagination typically starts at page 1.

The `test_zero_page` test demonstrates this bug by showing that when `page=0`, the returned `items` are the last 10 elements of the list (range(-10, 0)), instead of an empty list or an error.

**How to fix the bug:**

Add a check at the beginning of the function to ensure `page` is at least 1:

